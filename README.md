# API Security Risk Analysis

## Overview
This project demonstrates an API security assessment conducted on public demo APIs to identify common security risks such as broken authorization, weak authentication, excessive data exposure, and missing rate limiting.

## Tools Used
- Postman
- Chrome DevTools
- GitHub

## Security Areas Tested
- Authentication & Authorization
- Input Validation
- Rate Limiting
- Sensitive Data Exposure
- Security Headers

## Key Findings
- Broken Object Level Authorization (BOLA)
- Missing rate limiting
- Excessive data exposure
- Weak input validation

## Conclusion
The API security assessment identified several common security weaknesses, including authorization issues, missing rate limiting, and excessive data exposure. Implementing stronger access controls, input validation, and security protections can significantly improve the overall security posture of the API.
